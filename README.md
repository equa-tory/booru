# booru.local

A minimal Danbooru-style local photo gallery. No login, no cloud, no bullshit.

## Setup

```bash
pip install django pillow
python manage.py migrate
python manage.py runserver
```

Open http://localhost:8000

## Adding photos

**Option A — drop files into folder:**
Copy/move images (or whole subfolders) into `media/inbox/`, then click **⟳ scan inbox** in the browser.

**Option B — browser upload:**
Click **⬆ upload** and drag & drop files or pick them. Batch supported.

## AI tagging

### Quick (no ML deps, always works)
Out of the box, "ai tag all" falls back to tagging from filename tokens.

### Real AI tagging (recommended)
Install transformers + torch (CPU is fine for tagging):

```bash
pip install transformers torch torchvision
```

Then click **✦ ai tag all** — it uses CLIP (openai/clip-vit-base-patch32) to classify each
photo against ~25 labels (portrait, landscape, nature, architecture, food, animals...).
First run downloads ~350MB model, cached after that.

### WD14 tagger (Danbooru-style anime tags)
If you want actual Danbooru tags (1girl, blue_hair, etc.), install:

```bash
pip install onnxruntime huggingface-hub
```

Then in `gallery/views.py`, replace `run_ai_tagger()` with the WD14 version
(see comments at bottom of views.py).

## Tag categories & colors
| Category  | Color  | Use for                    |
|-----------|--------|----------------------------|
| general   | blue   | content descriptors        |
| character | pink   | character names            |
| artist    | yellow | artist/author names        |
| meta      | green  | file type, quality, etc.   |
| ai        | purple | auto-generated by AI       |

## Structure
```
media/
  inbox/    ← drop your photos here (subfolders OK)
  thumbs/   ← auto-generated, don't touch
db.sqlite3  ← all tags and metadata
```

Files are never moved or renamed. The DB only stores paths.
